Core Dependencies

Poppler (PDF Utilities)

Poppler is required for converting PDF files to images for OCR processing.

macOS: brew install poppler Requires Homebrew
Ubuntu/Debian: sudo apt-get install poppler-utils
Windows:

Download from poppler-windows and add the bin directory to your PATH

How to add to PATH in Windows
  1. Right-click on 'This PC' or 'My Computer' and select 'Properties'
  2. Click on 'Advanced system settings'
  3. Click on 'Environment Variables'
  4. Under 'System variables', find and select 'Path', then click 'Edit'
  5. Click 'New' and add the full path to the Poppler bin directory
  6. Click 'OK' on all dialogs to save changes

Verify installation by running pdftoppm -v in your terminal.

Tesseract OCR

Tesseract OCR Engine

Tesseract is the primary OCR engine used for text recognition.

macOS: brew install tesseract
For additional languages: brew install tesseract-lang
Ubuntu/Debian: sudo apt-get install tesseract-ocr
For additional languages: sudo apt-get install tesseract-ocr-all Or for specific languages (e.g., French): sudo apt-get install tesseract-ocr-fra
Windows:

Download installer from UB-Mannheim

Important: During installation, check "Add to PATH" option and select additional languages if needed.

Verify installation by running tesseract --version in your terminal.

Common Installation Issues
  • PATH issues: If 'command not found' occurs, ensure Tesseract is in your system PATH
  • Missing languages: Install additional language packs if you need non-English OCR
  • Windows DLL errors: Reinstall the Visual C++ Redistributable packages

Docker Installation (Alternative)

As an alternative to installing individual dependencies, you can use Docker to run the application with all dependencies pre-configured.

1. Install Docker
macOS:

Download Docker Desktop for Mac
Ubuntu/Debian: sudo apt-get install docker.io docker-compose
Windows:

Download Docker Desktop for Windows
2. Run with Docker

Clone the repository and run with docker-compose:

git clone https://github.com/fabriziosalmi/ocr-pdf-docx.git
cd ocr-pdf-docx
docker-compose up -d

This will start the application on port 8011, accessible at http://localhost:8011

Benefits of Docker Installation
  • No need to install Tesseract, Poppler, or other dependencies
  • Includes multiple language packs for Tesseract OCR
  • Consistent environment across different operating systems
  • Easier updates by pulling the latest image
Additional OCR Engines (Optional)

These additional OCR engines are optional and can be installed for specific use cases.

EasyOCR

All systems: pip install easyocr

EasyOCR may require additional system dependencies such as PyTorch. Refer to the EasyOCR documentation for details.

PyOCR

All systems: pip install pyocr
Requirements: Requires either Tesseract or Cuneiform to be installed.

Troubleshooting Common Issues

PDF Conversion Fails

Symptoms: "Poppler is not installed" error message or PDF conversion fails

Solutions:

  • Verify Poppler is installed correctly (run pdftoppm -v)
  • Check if Poppler is in your system PATH
  • Try reinstalling Poppler
  • Windows users: Make sure you've added the bin directory to PATH
Tesseract OCR Problems

Symptoms: "Tesseract is not installed" error or blank/incorrect text in output

Solutions:

  • Verify Tesseract is installed correctly (run tesseract --version)
  • Ensure the language packs you need are installed
  • Try using "high quality" mode for better results
  • For images with complex layouts, try using EasyOCR or PaddleOCR
Python Dependency Issues

Symptoms: Application fails to start or ModuleNotFoundError occurs

Solutions:

  • Reinstall Python dependencies with pip install -r requirements.txt
  • Consider using a virtual environment (python -m venv venv)
  • Check for version conflicts in installed packages
General Performance Issues

Symptoms: OCR is very slow or times out

Solutions:

  • Use "standard" quality instead of "high" for faster processing
  • Split large PDFs into smaller files
  • Increase memory allocation if using Docker
  • Ensure your system has adequate CPU and RAM resources available
